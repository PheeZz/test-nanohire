## 🎯 Задача
Создать **3 небольших сервиса** для системы подбора персонала.  
Всё должно быть понятно, без лишней сложности. Можно использовать моки (заглушки) вместо реальных внешних сервисов.

---

## 📦 Что нужно сделать

### 1️⃣ Основной API (для фронтенда)
**Что делает:**  
Сервис, с которым работает менеджер через интерфейс (фронтенд).

**Что реализовать:**
- ✅ Регистрация и вход (логин/пароль, JWT-токены)
- ✅ Вакансии: создать, посмотреть список, редактировать (только свои)
- ✅ Отклики: посмотреть отклики на вакансию, изменить статус («новый», «просмотрен», «отклонён»)
- ✅ Эндпоинт для сервиса уведомлений: — возвращает количество «новых» откликов
- ✅ Получает новые отклики из **RabbitMQ**

### 2️⃣ Сервис интеграции с HeadHunter
**Что делает:**  
Принимает «отклики» от HeadHunter и передаёт их в основной сервис.

**Что реализовать:**
- ✅ Эндпоинт `POST /webhook/hh` — принимает данные отклика. Принимать параметры аналогичные из https://api.hh.ru/openapi/redoc#tag/Webhook-API/operation/post-webhook-subscription -> Callbacks (NEW_RESPONSE_OR_INVITATION_VACANCY). В эндпоинт могут приходить дубли
- ✅ Проверяет заголовок `X-API-Key` (простая проверка: совпадает ли с секретом из .env)
- ✅ Реализовать псевдо-обращение к API HH для получение информации о резюме отклика (https://api.hh.ru/openapi/redoc#tag/Prosmotr-rezyume/operation/get-resume). Обрабатывать ошибки как на реальном запросе
- ✅ Отправляет данные в **RabbitMQ** (без дубликатов):
```
first_name - Имя
middle_name - Отчество
last_name - Фамилия
contacts - список контактов ([ { "type": "email", "value": "test@mail.com" } ])
```

### 3️⃣ Сервис ежедневных уведомлений
**Что делает:**  
Каждое утро в 8:00 отправляет менеджерам письма с напоминанием.

**Что реализовать:**
- ✅ Планировщик: запускает задачу ежедневно (пн - пт, без выходных) в 8:00 по Московскому времени
- ✅ Запрашивает у основного API: «Сколько новых откликов у каждого менеджера?»
- ✅ Формирует письмо:
  *«Здравствуйте, [Имя]! У вас [число] новых откликов, требующих обработки.»*
- ✅ Отправка: достаточно записать текст письма в консоль
